## Claude Code 검토

- A 모바일 필터 높이 축소를 최우선 채택했다. PR #10이 타깃을 키워 카드를 더 아래로 미는 만큼 상보 변경이 필요하다고 봤다.
- B 로딩·오류·재시도 상태, C 대회별 접근 가능한 이름과 선택 상태, D 알림 저장 후 모달 자동 닫기, F 마감 시각 우선 판정과 테스트를 채택했다.
- E `content-visibility`는 실측 성능 병목이 없고 스크롤 점프 위험이 있어 이번 범위에서 보류했다.
- 서비스워커의 HTML 폴백을 탐색 요청으로 제한하고 `saveJson` 쓰기 예외를 처리하는 작은 안정성 보완을 권고했다. 서비스워커 전략 전체 개편과 백그라운드 재무장은 보류했다.
- 모바일 A의 구체안으로 제목 축소만으로도 충분할 수 있다고 제안했다.
- 리드 예비 반박: PR #10 적용 실측에서 첫 알림 CTA 상단은 y=639, 하단 탭 상단은 y=485다. 제목 축소만으로 필요한 약 198px를 줄일 수 없어 고급 필터와 거리필터의 기본 접기 또는 동등한 구조적 압축이 필요하다.
- 320px 가로 넘침은 `.content-grid` 안 `.race-list`의 intrinsic 최소 폭이 320px로 계산되는 것이 원인이다. 컨테이너는 295px라 `min-width: 0` 보정이 필요하다.

## 담당자 교차 검토

- 모바일 UX는 제목 축소만으로 약 198px의 격차를 줄일 수 없다고 반박했다. 지역·검색 고급 필터는 기본 접기하고 현재 조건을 버튼에 요약하며, `min-width: 0`, 로딩 상태, 접근 가능한 이름, 성공 후 모달 닫기를 채택했다.
- 접근성은 거리 필터까지 숨기는 안에 반대했다. 카테고리와 거리는 항상 노출하고 지역·검색만 `aria-expanded` 토글 아래 두며, 필터 재렌더 후 포커스와 결과 갱신 전달을 보완해야 한다고 봤다.
- PWA·성능도 거리 필터 접기에 반대하고, 지역·검색 접기와 모바일 여백 축소 후 재측정을 제안했다. 서비스워커는 탐색 전용 HTML 폴백보다 요청 유형별 전략까지 넓힐 것을 제안했으나 리드는 회귀 위험 때문에 이번에는 폴백 오류만 고친다.
- 안정성은 지역·검색과 거리 필터를 함께 접는 안을 지지했지만, 숨겨진 조건을 반드시 요약해야 한다고 보완했다. 저장 성공일 때만 모달을 닫고, 마감 시각 판정은 CI 경계 테스트와 묶어야 한다고 강조했다.
- 네 담당자 모두 `content-visibility`와 전역 고정 대회 CTA는 보류하고, 320px 넘침·필터 높이·오류 상태·ARIA·마감 시각 판정을 우선하는 데 합의했다.

## 리드 최종 범위

### 채택

1. PR #10의 44px 타깃 변경을 보존하고 320px 가로 넘침을 `min-width: 0`으로 제거한다.
2. 모바일에서 카테고리와 거리 필터는 노출하되, 지역·검색·확인만 기본 접기한다. 접힌 버튼에는 현재 지역·검색 조건을 요약한다.
3. 모바일 거리 필터는 44px 한 줄 가로 스크롤로 바꿔 수직 공간을 줄인다. 제목과 여백은 기존 시각 언어를 유지하는 범위에서만 축소한다.
4. 로딩·오류·빈 결과를 분리하고 오류 상태에 재시도 버튼과 `aria-busy`를 제공한다.
5. 카드 `접수·알림`에 대회명을 포함한 접근 가능한 이름을 주고, 내비·카테고리·거리 선택 상태를 프로그램적으로 전달한다.
6. 알림 저장 성공 후 모달을 닫는다. 저장 실패·선택값 없음·권한 처리 실패 경로에서는 닫지 않는다.
7. 마감 시각이 지난 `status: open` 대회를 접수중으로 보지 않도록 순수 로직과 런타임 정규화를 고치고 경계 테스트를 추가한다.
8. 서비스워커 HTML 폴백을 탐색 요청으로 제한하고, JSON 저장 실패를 사용자에게 알릴 수 있게 쓰기 가드를 추가한다.
9. 모달 닫기·프리셋 타깃을 44px로 맞추고 작은 화면에서 내부 스크롤을 허용한다.

### 보류·제외

- 전역 고정 대회 CTA는 대상 대회 맥락이 불명확하고 하단 내비와 경쟁하므로 제외한다.
- `content-visibility`, 서비스워커 전략 전면 개편, 앱 복귀 알림 재무장, 서버 Web Push는 실측 또는 별도 정책이 필요한 범위라 보류한다.
- 외부 HTTP 링크 176개 정리는 데이터 출처별 확인이 필요한 운영 작업이라 이번 UI·안정성 PR에서 제외한다.

## 구현 후 실측

- PR #10 적용 전 main의 320×568 첫 카드는 y=538, 접수는 y=561~605, 알림은 y=635~679였다.
- 최종 구현의 320×568 첫 카드는 y=325, 접수는 y=356~400, 알림은 y=447~491이다.
- 첫 카드는 213px, 알림 CTA 상단은 188px 올라갔고 두 행동 모두 하단 내비 y=500 위에 완전히 노출된다.
- 320px 가로 넘침은 `true`에서 `false`로 바뀌었다.
- 모바일 필터를 펼치면 지역·검색·확인이 다시 표시되고, 적용 뒤 접힌 버튼에 선택 지역이 요약된다.
- 모달 닫기와 프리셋은 모두 44px이며, 모달 높이는 457px·내부 스크롤 가능 상태다.
- 360×800과 1280×800에서도 가로 넘침과 오류 오버레이가 없고 데스크톱 필터 배치는 유지된다.
