# Changelog

PushRun(러닝 대회 접수 알림)의 주요 변경 이력. [SemVer](https://semver.org/lang/ko/).

## [0.6.10] - 2026-07-11
정식 공개 준비 — 좁은 폰 화면 레이아웃 깨짐과 아이콘·보안을 손봤다.

### Fixed
- **접수 버튼 글자 깨짐**: 좁은 폰에서 "접수 · HTTP" 라벨이 넘쳐 카드가 깨지던 문제를 고쳤다.
  버튼에는 항상 "접수"만 표시하고, HTTP 보안 경고는 화면 폭을 차지하지 않는 보조 라벨(툴팁·aria)로만 유지한다.
- **제목 밑줄 어긋남**: "대회 신청 접수 알리미" 제목 밑줄이 "대회 신청"까지만 어중간하게 걸리던 것을
  제목 전체 폭에 맞게 고쳤다.
- **지역·장소 pill 넘침**: 장소명이 긴 대회(예: "경북 · 김천종합스포츠타운")에서 pill 텍스트가
  카드를 넘겨 깨지던 것을 말줄임(…) 처리했다.
- **D-day 컬럼 넘침**: 3자리 D-day("D-120")·"일정 대기"가 좁은 날짜 컬럼을 넘지 않게 폰트 하한을 조정했다.

### Added
- **iOS·안드로이드 앱 아이콘**: `apple-touch-icon.png`(180) + `icon-192/512.png` 를 추가하고
  매니페스트·서비스워커·head 메타(apple-mobile-web-app-*)에 연결했다. 이제 아이폰 "홈 화면에 추가" 시
  페이지 스크린샷이 아니라 PushRun 브랜드 아이콘이 표시된다.

### Security
- 접수 링크에 `http(s)` 스킴 화이트리스트 검증을 추가해 외부 데이터의 `javascript:`/`data:` 스킴을 차단(방어적 하드닝).

## [0.6.9] - 2026-07-11
알림 신뢰성(최신 데이터 기준 재계산)과 배포 검증을 강화한 출시 준비판.

### Fixed
- **알림 시각을 최신 대회 데이터로 다시 계산**: 알림을 켠 뒤 races.json 에서 접수 시각이 바뀌어도
  예전 시각에 울리던 문제를 고쳤다. 앱을 열거나 새로고침할 때마다 저장된 알림의 발사 시각을
  현재 데이터 기준으로 재계산하고, 사라진 대회의 알림(고아)과 발사 시각이 이미 지난 알림(만료)은
  즉시 정리한다 — '내 알림'에는 실제로 울릴 알림만 남는다.
- **서비스워커 캐시 저장 보장**: 캐시 쓰기를 `event.waitUntil` 에 묶어 워커가 먼저 종료돼도
  저장이 유실되거나 처리되지 않은 rejection 이 생기지 않게 했다 (네트워크 우선 동작은 그대로).
- `venue`/`time`/`distances` 가 빠진 대회 항목 1개 때문에 앱 전체가 하얗게 죽던 지점에 방어 코드를 넣었다.
- 알림 목록 버튼의 `race.id` 를 escapeHtml 로 이스케이프하고, `cafe.naver.com` 접수 링크 2건을 https 로 바꿨다.

### Added
- `alerts-core.js`: 알림 핵심 로직(발사 시각 계산·재조정·만료 정리·24.8일 클램프)을 순수 함수로 분리.
- `scripts/alerts-core.test.mjs`: 시각 변경 재계산, 고아 제거, 만료 정리, 재무장 클램프, 깨진 데이터
  내성까지 검증하는 node:test 동작 테스트. `npm test` 에 포함.
- validator 강화: 대회 필수 필드(venue·time·distances 등) 검사, 데이터 신선도 게이트(접수 예정 0개
  또는 종료 대회 50% 초과 시 FAIL), sw.js APP_SHELL 캐시버스트·CACHE_NAME↔package.json 버전 일치 검사.
- 설정 화면에 "알림 권한 안내" 버튼을 추가해 닿을 수 없던 권한 안내 모달을 복원하고,
  권한을 이미 차단한 경우 브라우저 사이트 설정에서 다시 켜는 방법 안내를 더했다.
- `.github/workflows/daily-self-improve.yml`: 하루 2회(KST 06:00/18:00) 자가개선 draft PR 자동화.
- `npm start` 를 크로스플랫폼 정적 서버(`scripts/serve.mjs`, 의존성 없음)로 교체.

### Removed
- 존재하지 않는 DOM 을 그리던 죽은 코드(`renderDetail`·`raceSectionHtml`·`raceCountLabel`)와
  1초 타이머의 불필요한 호출, 오래된 개인 인수인계 문서(`codex-handoff/`)를 삭제했다.

## [0.6.8] - 2026-07-10
출시 전 알림 안내·PWA·접근성·배포 검증을 보강했다.

### Fixed
- 브라우저 페이지가 닫힌 뒤에도 알림이 보장되는 것처럼 안내하던 배터리 설정 문구와 Android intent 이동을 제거하고, 현재 웹 타이머의 실제 동작 범위를 명확히 표시했다.
- 서비스워커가 있으면 `showNotification`을 사용하고 실패 시 화면 토스트를 유지해 모바일 알림 표시 실패가 앱 실행을 중단하지 않도록 했다.
- 모달을 열 때 첫 요소로 포커스를 이동하고 Tab 순환·닫은 뒤 포커스 복원을 추가했다.
- HTTP 외부 접수 링크에 연결 주의 문구를 표시했다.

### Added
- PWA manifest, 아이콘, 앱 셸 서비스워커를 추가했다.
- 100개 대회 데이터의 필수값·날짜·중복·URL·버전·캐시버스트를 검사하는 `npm test`를 추가했다.
- PR에서도 검증을 실행하고, main 배포는 검증 성공 후에만 진행하도록 Pages 워크플로를 변경했다.

## [0.6.7] - 2026-07-09
알림이 조용히 안 울리던 문제 3건 수정 (앱 핵심 가치).

### Fixed
- **먼 미래 대회 알림이 아예 예약되지 않던 문제**: `setTimeout` 최대 지연(약 24.8일)을 넘는 알림을 그냥
  버려서, 25일 이상 남은 대회(8·9월 등)는 알림을 켜도 영영 안 울렸다. → 상한만큼 자고 전체 스케줄을 다시
  거는 "재무장"으로 교체(`scheduleBrowserTimers`).
- **알림 모달의 체크박스가 매초 초기화되던 문제**: 모달이 열려 있는 동안 1초 타이머가 모달 전체를 다시
  그려서, 사용자가 방금 바꾼 오프셋 선택(예: "10분 전" 해제)이 1초 만에 기본값으로 되돌아갔다. → 매초 갱신은
  카운트다운만 하도록 분리(`renderModalCountdown`).
- **데이터 갱신 시 저장된 알림이 사라지던 문제**: 스케줄 피드 항목 ID가 배열 index 기반이라 피드가
  재정렬·삽입되면 ID가 바뀌어 localStorage 의 알림이 전부 고아가 됐다. → 내용(이름+날짜) 기반 안정 ID로 교체.

### Changed
- 캐시버스트 갱신: `ASSET_VERSION`/`?v=` 20260708-9 → 20260709-1 (app.js·index.html·package.json 일치) —
  고친 내용이 사용자 브라우저에 실제 반영되도록.
