# Changelog

## [Unreleased]

### Changed

- 운영체제 다크 모드에서도 따뜻한 크림·코랄 톤과 읽기 쉬운 접수 버튼 대비를 유지한다.

## [0.7.0] - 2026-07-12

### Changed

- 실제 대회 데이터에서 가장 가까운 접수 일정을 고르는 히어로와 이번 주 접수 목록, 홈·알림·설정 하단 내비로 모바일 화면을 전면 재디자인했다.
- 다른 로봄 앱, 문의·광고 제휴, 정책·라이선스, 개발자·앱·데이터 버전을 설정에 통합했다.
- `pushrun:alert-subscriptions:v3`, `PushRunAlertsCore`, 안정 ID와 종목별 `targetKey`는 유지하고 새 정적 자산이 갱신되도록 버전·캐시를 동기화했다.

## [0.6.17] - 2026-07-12

### Added
- 날짜만 확인된 접수도 시간 확인 알림을 켤 수 있다. 기본값은 접수일 자정·09:00·12:00·18:00이며 필요한 시각만 선택할 수 있다.
- 데이터에 정확한 접수 시간이 추가되면 기존 시간 확인 알림을 접수 전·정각 정확 알림으로 자동 전환한다.

### Changed
- 제23회 경남마라톤 접수 시작을 공식 홈페이지 기준 7월 13일 10:00으로, WYD 생명사랑 마라톤을 공개 대회 안내 기준 8월 2일 09:00으로 갱신했다.
- 시간 미확정 알림은 실제 접수가 열렸다고 단정하지 않고 대회 사이트의 공식 접수 여부를 확인하도록 안내한다.
- 캐시버스트를 `20260712-4`, 서비스워커 캐시를 `pushrun-v0.6.17`로 갱신했다.

## [0.6.15] - 2026-07-12

### Added
- Full·10K처럼 접수 시각이 다른 종목을 알림 모달에서 복수 선택하고 각각 저장할 수 있다.
- 내 알림 화면에서 종목별 접수 알림을 따로 확인하고 개별 해제할 수 있다.
- 기존 대회 단일 알림을 일치하는 종목 알림으로 한 번만 승격하는 호환 처리를 추가했다.

### Changed
- 카드의 접수 일정을 `종목 | 날짜 | 시간` 행으로 분리해 종목과 접수 시각의 대응을 명확히 했다.
- 모바일 접수·알림 CTA를 카드 하단 2열 44px 버튼으로 배치해 접수 일정 폭과 터치 편의성을 확보했다.
- 종목별 구독은 선택한 일정이 만료돼도 다른 종목으로 자동 전환하지 않고 해당 알림만 정리한다.
- 캐시버스트를 `20260712-2`, 서비스워커 캐시를 `pushrun-v0.6.15`로 갱신했다.

## [0.6.12] - 2026-07-11

### Fixed

## [0.6.11] - 2026-07-11
모바일 첫 화면의 핵심 행동 접근성과 데이터·오프라인 실패 처리를 개선했다.

### Changed
- 320px 화면에서 지역·검색 필터를 기본 접고 거리 필터를 한 줄로 유지해 첫 대회 카드의 접수·알림 행동을 위로 올렸다.
- 모바일 필터 버튼에 현재 지역·검색 조건을 요약하고, 적용 뒤 자동으로 결과 화면을 넓게 보여준다.
- 대회별 접수·알림에 구체적인 접근 가능한 이름을 추가하고 내비·카테고리·거리 선택 상태를 전달한다.

### Fixed
- 320px에서 대회 목록의 intrinsic 최소 폭 때문에 가로 스크롤이 생기던 문제를 고쳤다.
- 로딩·통신 오류·정상 빈 결과를 구분하고 오류 상태에서 바로 다시 불러올 수 있게 했다.
- 알림 저장 성공 뒤 모달이 남아 닫기 버튼까지 다시 손을 옮겨야 하던 흐름을 고쳤다.
- `status: open`이어도 접수 마감 시각이 지났으면 접수중으로 표시하지 않도록 수정하고 경계 테스트를 추가했다.
- 서비스워커가 JS·JSON 캐시 실패에 HTML 셸을 반환하던 폴백을 탐색 요청으로 제한했다.
- 브라우저 저장소 쓰기 실패 시 알림 상태가 성공한 것처럼 남지 않게 했다.
- 작은 화면 모달에 내부 스크롤을 허용하고 닫기·알림 프리셋 타깃을 44px로 맞췄다.

## [0.6.10] - 2026-07-11
정식 공개 준비 — 좁은 폰 화면 레이아웃 깨짐과 아이콘·보안을 손봤다.

### Fixed
- **접수 버튼 글자 깨짐**: 좁은 폰에서 "접수 · HTTP" 라벨이 넘쳐 카드가 깨지던 문제를 고쳤다.
  버튼에는 항상 "접수"만 표시하고, HTTP 보안 경고는 화면 폭을 차지하지 않는 보조 라벨(툴팁·aria)로만 유지한다.
- **제목 밑줄 어긋남**: "대회 신청 접수 알리미" 제목 밑줄이 "대회 신청"까지만 어중간하게 걸리던 것을
  제목 전체 폭에 맞게 고쳤다.
- **지역·장소 pill 넘침**: 장소명이 긴 대회(예: "경북 · 김천종합스포츠타운")에서 pill 텍스트가
  카드를 넘겨 깨지던 것을 말줄임(…) 처리했다.
- **D-day 컬럼 넘침**: 3자리 D-day("D-120")·"일정 대기"가 좁은 날짜 컬럼을 넘지 않게 폰트 하한을 조정했다.

### Added
- **iOS·안드로이드 앱 아이콘**: `apple-touch-icon.png`(180) + `icon-192/512.png` 를 추가하고
  매니페스트·서비스워커·head 메타(apple-mobile-web-app-*)에 연결했다. 이제 아이폰 "홈 화면에 추가" 시
  페이지 스크린샷이 아니라 PushRun 브랜드 아이콘이 표시된다.

### Security
- 접수 링크에 `http(s)` 스킴 화이트리스트 검증을 추가해 외부 데이터의 `javascript:`/`data:` 스킴을 차단(방어적 하드닝).

## [0.6.9] - 2026-07-11

  예전 시각에 울리던 문제를 고쳤다. 앱을 열거나 새로고침할 때마다 저장된 알림의 발사 시각을
  현재 데이터 기준으로 재계산하고, 사라진 대회의 알림(고아)과 발사 시각이 이미 지난 알림(만료)은

  내성까지 검증하는 node:test 동작 테스트. `npm test` 에 포함.

## [0.6.7] - 2026-07-09
알림이 조용히 안 울리던 문제 3건 수정 (앱 핵심 가치).

### Fixed
- **먼 미래 대회 알림이 아예 예약되지 않던 문제**: `setTimeout` 최대 지연(약 24.8일)을 넘는 알림을 그냥
  버려서, 25일 이상 남은 대회(8·9월 등)는 알림을 켜도 영영 안 울렸다. → 상한만큼 자고 전체 스케줄을 다시
  거는 "재무장"으로 교체(`scheduleBrowserTimers`).
- **알림 모달의 체크박스가 매초 초기화되던 문제**: 모달이 열려 있는 동안 1초 타이머가 모달 전체를 다시
  그려서, 사용자가 방금 바꾼 오프셋 선택(예: "10분 전" 해제)이 1초 만에 기본값으로 되돌아갔다. → 매초 갱신은
  카운트다운만 하도록 분리(`renderModalCountdown`).
- **데이터 갱신 시 저장된 알림이 사라지던 문제**: 스케줄 피드 항목 ID가 배열 index 기반이라 피드가
  재정렬·삽입되면 ID가 바뀌어 localStorage 의 알림이 전부 고아가 됐다. → 내용(이름+날짜) 기반 안정 ID로 교체.

### Changed
- 캐시버스트 갱신: `ASSET_VERSION`/`?v=` 20260708-9 → 20260709-1 (app.js·index.html·package.json 일치) —
  고친 내용이 사용자 브라우저에 실제 반영되도록.
