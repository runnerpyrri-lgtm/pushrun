# Changelog

러닝봄(러닝 대회 접수 알림)의 주요 변경 이력. [SemVer](https://semver.org/lang/ko/).

## 0.17.13 (2026-07-20)

- 대회 접수 링크(등록 URL·출처 상세 URL) 헬스체크를 추가했다. 기존 `validate-static.mjs`는 URL 형식(파싱 가능 여부, https)만 검사하고 실제로 링크가 살아 있는지는 확인하지 않았다 — 접수처가 도메인을 바꾸거나 페이지를 내려도 정적 검증은 통과했다.
- `scripts/link-healthcheck-core.mjs`: 네트워크 없이 오프라인 단위 테스트가 가능한 순수 로직(races.json에서 링크 추출·중복 제거, URL 파싱, HTTP 상태 코드로 생사 판정, 리다이렉트 감지·집계).
- `scripts/link-healthcheck.mjs`: 실제 HEAD/GET 요청으로 모든 접수 링크를 점검하는 CI 전용 스크립트(`npm run linkcheck`). 죽은 링크가 있으면 실패 종료하고 `outputs/link-healthcheck-report.json`에 상세 리포트를 남긴다. 네트워크가 필요해 오프라인 `npm test`에는 포함하지 않았다.
- `scripts/link-healthcheck-core.test.mjs`: 위 순수 로직에 대한 `node --test` 단위 테스트(URL 파싱, 상태 코드 판정, 리다이렉트 감지, 링크 추출·중복 제거, 집계).

## 0.16.0 (2026-07-14)

메인(대회 찾기) 화면을 확정 시안대로 전면 재설계했다. 헤더·로고·글자체와 알림 저장 키·대회 ID·서비스워커·PWA 설치 경로는 그대로 유지한다.

- **검색 + 지역 한 줄**: 큰 검색창(52px) 옆에 네이티브 지역 `<select>`를 두고, 적용 버튼 없이 고르면 즉시 목록이 갱신된다. 지역 칩 나열·별도 지역 패널을 제거했다.
- **거리 필터 2행 × 3열 고정 그리드**: 전체·풀코스·하프 / 10km·5km·트레일. 어떤 해상도(320~768px)에서도 가로 스크롤이 없고 트레일이 잘리지 않는다. 선택은 남색 배경+흰 글자+`aria-pressed`.
- **접수 상태 3분할**: 전체 / 접수 중 / 접수 예정. 전체 = 접수 중 + 접수 예정을 한 목록에 함께 보여주고, 각 개수는 현재 지역·거리·검색 조건을 반영해 재계산된다. 마감 대회는 기본 제외, `조건 더보기 → 접수 마감 포함` 스위치를 켤 때만 표시한다.
- **결과 요약 + 정렬**: `총 N개`와 조건 보조줄, 정렬 `<select>`(대회일 빠른 순 기본 / 접수일 빠른 순 / 지역순). 필터가 걸리면 밑줄 텍스트 `전체 초기화`가 나타난다.
- **세로 카드 목록**: 상단 상태 색선 + 텍스트 배지(접수 중 코랄 / 접수 예정 스카이·시간 확정 민트 / 시간 미확정 슬레이트 점선 / 마감 회색), 대회명(최대 3줄, 말줄임 없음)·대회일·지역·거리 태그·가장 중요한 접수 일정 한 줄. 강한 코랄 CTA는 접수 예정이면 `알림 설정`, 이미 접수 중이면 `지금 접수하기`(공식 접수처)로 상황에 맞춘다.
- **자세히 보기 아코디언**(한 번에 한 카드): 출발 장소, 네이버지도 버튼(출발지 미검증 시 비활성 안내), 접수 일정, 종목별 접수 시각, 대회 일정, 주최·규모, 공식 접수처, 데이터 출처 순.
- 지도 링크는 순수 함수(`compactMapText`·`raceLocationSearchCandidates`·`naverMapSearchUrl`·`raceMapLink`)로 분리하고 Node 테스트를 추가했다(https 스킴·URL 인코딩·후보 중복 제거·출발지 없을 때 null). 기존 알림·정렬·캘린더 로직과 테스트는 그대로 통과(총 33개).
- 캐시 `pushrun-v0.16.0`/자산 `20260714-01`.
